# [FR-004] Nudge Decision Engine

## Labels
`MUS`, `enhancement`, `coaching`

## User Story
As a founder, I want context-aware nudges generated when my activity indicates drift, stagnation, or over-fragmentation so that I can course-correct quickly.

## Proposed Solution
### Overview
Implement rules-driven nudge generation triggered by score thresholds and behavior patterns.

### Implementation Flow
1. Define nudge rules and cooldown constraints.
2. Evaluate recent `ProjectSignal` + `FocusScore` windows.
3. Create `Nudge` records with reason text and target channels.

### Technical Approach
```python
if score < config.low_focus_threshold and switches > config.switch_limit:
    emit_nudge(type="prioritize", severity="high", reason="low focus + high context switching")
```

### Key Considerations
- Avoid alert fatigue via cooldowns and quiet hours.
- Every nudge must include machine-generated reason text.

## Acceptance Criteria
- [ ] Nudges are generated when rule thresholds are crossed.
- [ ] Cooldown prevents repeated identical nudges inside configured window.
- [ ] Each nudge contains a user-visible reason string.
