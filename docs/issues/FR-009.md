# [FR-009] Smart Categorization Presets

## Labels
`Future`, `enhancement`, `onboarding`

## User Story
As a founder, I want optional presets for classifying common startup work modes so that setup time is faster.

## Proposed Solution
### Overview
Provide preset bundles (e.g., Product Build, Fundraising, Growth Sprint) that pre-fill signal weights and rule thresholds.

### Implementation Flow
1. Define preset templates mapped to tracker config values.
2. Add preset selector in onboarding/settings.
3. Allow user preview + one-click apply.

### Technical Approach
```ts
const PRESETS = {
  productBuild: { signalWeights: { code: 1.0, research: 0.4 }, nudgeSensitivity: 'medium' }
}
```

### Key Considerations
- Presets should be editable after apply.
- Keep presets opinionated but transparent.

## Acceptance Criteria
- [ ] At least three presets are available and documented.
- [ ] Applying a preset updates config fields consistently.
- [ ] Founder can revert preset changes before saving.
