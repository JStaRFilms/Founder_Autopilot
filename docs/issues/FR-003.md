# [FR-003] Focus Scoring Engine

## Labels
`MUS`, `enhancement`, `scoring`

## User Story
As a founder, I want daily and rolling focus scores computed from my activity signals so that I can quantify deep work versus drift.

## Proposed Solution
### Overview
Implement a scoring job that aggregates normalized events into `ProjectSignal` windows and outputs `FocusScore` values.

### Implementation Flow
1. Build signal extraction logic for focus/drift/context switches.
2. Apply configurable weighting from `TrackerConfig`.
3. Persist daily score and 7-day trend direction.

### Technical Approach
```python
score = clamp(
    base + (focus_minutes * weight_focus) - (drift_minutes * weight_drift) - (switches * weight_switch),
    0,
    100,
)
```

### Key Considerations
- Score must be explainable with contributing factors.
- Re-runs should be idempotent for the same date window.

## Acceptance Criteria
- [ ] Focus scores generated for each day with at least one event.
- [ ] Score output includes contributing signal breakdown.
- [ ] Recomputing same window yields identical score values.
