# [FR-001] Local Activity Ingestion Daemon

## Labels
`MUS`, `enhancement`, `backend`

## User Story
As a founder, I want the system to collect activity events from Git, COS, and configured folders automatically so that I do not manually log work.

## Proposed Solution
### Overview
Implement a long-running local daemon that polls or subscribes to source-specific adapters and writes raw events to SQLite.

### Implementation Flow
1. Build source adapters for Git, COS, and filesystem watchers.
2. Add ingestion scheduler with deduplication guardrails.
3. Persist raw events with source-specific metadata.

### Technical Approach
```python
class IngestionDaemon:
    def run(self):
        for adapter in self.adapters:
            events = adapter.collect(since=self.last_cursor[adapter.name])
            self.repo.save_raw_events(events)
```

### Key Considerations
- Must be local-only and resilient to restarts.
- Prevent duplicate events across polling windows.

## Acceptance Criteria
- [x] Daemon captures events from all configured sources without manual trigger. Completed via default Git, COS, and filesystem adapters wired into `DaemonService`.
- [x] Restarting daemon resumes from last processed cursor. Completed via `source_cursors` persistence and resume-aware adapter filtering.
- [x] Duplicate event rate stays below 1% across a 24h run. Completed via per-source raw-event checksum dedupe and regression coverage.
