# [FR-005] Multi-Channel Notification Delivery

## Labels
`MUS`, `enhancement`, `notifications`

## User Story
As a founder, I want nudges delivered to Codex inbox, dashboard notification center, and OS notifications so that I reliably see guidance where I already work.

## Proposed Solution
### Overview
Create a delivery service that fans out pending `Nudge` items across enabled channels and records delivery status.

### Implementation Flow
1. Implement channel adapters for Codex, dashboard feed, and OS notifications.
2. Add retry strategy with backoff per channel.
3. Store per-channel delivery receipts.

### Technical Approach
```python
for channel in nudge.target_channels:
    receipt = adapters[channel].deliver(nudge)
    repo.record_delivery(nudge.id, channel, receipt.status)
```

### Key Considerations
- Channel failures should not block other channels.
- Respect quiet hours for intrusive OS notifications.

## Acceptance Criteria
- [ ] A single nudge can be delivered to all enabled channels.
- [ ] Failed channel deliveries are retried and logged.
- [ ] Dashboard always reflects latest nudge delivery status.
