# [FR-002] Canonical Event Normalization

## Labels
`MUS`, `enhancement`, `data-model`

## User Story
As a founder, I want all captured activities converted into a consistent event schema so that downstream scoring and reporting remain reliable.

## Proposed Solution
### Overview
Create a normalization pipeline that maps source events into `ActivityEvent` and validates required fields.

### Implementation Flow
1. Define canonical schema contract and validators.
2. Build mapping functions per source adapter.
3. Store normalized events in dedicated table and log invalid rows.

### Technical Approach
```python
def normalize(raw_event: dict) -> ActivityEvent:
    mapped = mapper_for(raw_event["source"])(raw_event)
    validate_activity_event(mapped)
    return mapped
```

### Key Considerations
- Invalid events should not crash ingestion.
- Retain traceability from normalized to raw event ids.

## Acceptance Criteria
- [x] 100% of persisted normalized rows pass schema validation. Completed via deterministic per-source normalization before `activity_events` writes.
- [x] Invalid events are quarantined with error reason and source id. Completed via `invalid_events` persistence on normalization failure.
- [x] Normalization is deterministic for identical raw input. Completed via stable UTC timestamp normalization, deterministic IDs, and checksum-based raw dedupe.
