# [FR-008] Tracker Configuration Controls

## Labels
`MUS`, `enhancement`, `settings`

## User Story
As a founder, I want to configure watch paths, signal weighting, quiet hours, and nudge aggressiveness so that the system matches my workflow.

## Proposed Solution
### Overview
Create settings storage and UI/editor for `TrackerConfig` with validation and safe defaults.

### Implementation Flow
1. Define persisted config schema and defaults.
2. Implement settings API + dashboard form.
3. Hot-reload daemon/scoring config without full reinstall.

### Technical Approach
```python
def update_tracker_config(new_config):
    validated = validate_tracker_config(new_config)
    repo.save_config(validated)
    runtime.reload(validated)
```

### Key Considerations
- Invalid config changes must be rejected with clear error messages.
- Config changes should take effect within one polling cycle.

## Acceptance Criteria
- [ ] Founder can edit watch paths, weights, quiet hours, and sensitivity in UI.
- [ ] Invalid configuration is blocked with field-level feedback.
- [ ] Updated config is persisted and applied without data loss.
